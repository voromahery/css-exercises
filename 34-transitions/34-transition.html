Transition

Definition:

The word transition means to change from one state to another. In CSS, these transitions occur per element, per property

  1. The first state is the initial state, or how the element was to begin with.
  2. The second state, or target state, is what the element will become. Transitions take care of all the intermediate stages between those two states.

Research:

  1. The transition property is a shorthand property which includes transition-property, transition-duration, timing-function and delay.
  2. The difference between timing function, delay and duration:
    • timing function is a function which defines the speed of an animated object.
    • delay defines how many seconds or milliseconds it takes for an animated object to wait before it animated.
    • duration specifies how many seconds and milliseconds it takes for an animated object to complete from the starting point and the end.

  4. We can select more than one property to transition at a time by using transition: propery, duration, delay, timing-function for all of them, comma separated and all keyword.
  5. If we want to set or define the desired transition, we should use pseudo-classes, active, :hover, :target, :checked, :focus selector which is different from anothers like using class and id.
  6. We need transitions especially when we are telling story, to show the progress of one thing.
  7. Transitions can be a problem for something that we need urgently.
  8. Transitions don't work reliably with the auto keyword being one of the end states because the browsers need number to calculate but auto isn't a number.
  9. To stagger transitions (i.e. make a number of child elements all start at different times), we have to make the value of delay different.
