Custom Properties

  1. When defining a Custom Property's name, we have to use -- or double dashes as a prefix, alpha-numeric and number.
  2. We can't use space, period(.).
  3. We can use numbers or a number with unit, keywords such as color annotations and variation of fonts to assign to a new Custom Property.

  4. It is useful to create Custom Properties using the :root or html if we want each child element inherit all of the HTML property value of the parent element.

    It isn't useful when we want to change the property value of an element.

  5. After we've defined a new Custom Property, we can use it as a value of another properties in a function.

    The function that we use is var(custom property name).

  6. The ways that override or change a Custom Property are:
    • Using @media or media query
    • Using selector basis
    • Editing or changing the value
  7. Custom Properties are useful because they help us to:
    • write less code
    • reduce repetition
    • one point of control
  8. Declaration block is the only aspect (or part) of CSS syntax that we can use Custom Properties.
