Overflow and clipping

Introduction

Overflow in CSS refers to the way the contents of a box are handled if they don't fit within that box.

When a width restraint is placed on a box, the part of the text and other content within that box will go out of the box.

Properties

overflow: visible

It helps to manage the content in a box if the content is too big for the box so that it can fit properly in a box. It helps to control the layout.

To make it work, we need border.

clip: auto

It controls what part of a content is visible. It can also help to position an element.

To make it work, we need border.

clip-path: auto

It creates clipping region that sets what part of an element should be shown. It allows us to make complex shape by clipping an element into a basic shape (circle, polygon, ellipse...).

To make it work, we need clip-path.

text-overflow: ellipsis

It decreases the amount of text displayed.

To make it work, we need padding and border.

The browser's default way to handle contents that larger than its container is a overflow-container.

Ellipsis displays ellipsis to present clipped text. However, ellipse is a way to clip a content to make it circular.

text-overflow is different from other properties because it only affects content that is overflowing a block container element in its inline progression direction.
